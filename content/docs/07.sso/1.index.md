>在物理或虚拟服务器上开始使用Keycloak。确保您的机器或容器平台已安装OpenJDK 21且能够提供足够的内存和CPU，以满足 Keycloak 的预期使用需求。有关如何开始进行生产环境规模调整的更多信息，请参阅[CPU 和内存资源规模调整概念](https://www.keycloak.org/high-availability/single-cluster/concepts-memory-and-cpu-sizing)。
## 下载安装

### 下载安装包

到[官网](https://www.keycloak.org/downloads)下载安装包

```plain
wget https://github.com/keycloak/keycloak/releases/download/26.4.1/keycloak-26.4.1.zip
# 或
wget https://github.com/keycloak/keycloak/releases/download/26.4.1/keycloak-26.4.1.tar.gz
```
### 安装到系统

```plain
tar -zxvf keycloak-25.0.6.tar.gz
mv keycloak-25.0.6 /opt/keycloak
```
## 编辑配置

配置文件在`conf/keycloak.conf`，此方式是命令行形式指定参数，也可以环境变量形式指定，如`/opt/keycloak/env/keycloak.env`

### 开发环境

不需要生成并配置HTTPS证书

```plain
# /opt/keycloak/conf/keycloak.conf
# 数据库
db=mysql
db-username=root
db-password=123456
db-url-host=127.0.0.1
db-url-port=3306
db-url-database=keycloak_db
db-pool-max-size=10
db-pool-min-size=5
# 管理员
bootstrap-admin-username=admin
bootstrap-admin-password=admin
# 服务
http-host=0.0.0.0
http-port=9080
# 有代理时用
# proxy-headers=xforwarded
http-enabled=true
hostname-strict=false

health-enabled=true
metrics-enabled=true


log=console
log-level=info

http-management-port=9090
http-management-health-enabled=true
```
### 生产环境

需要生成并配置HTTPS证书

```yaml
db=mysql
db-username=root
db-password=City#2025
db-url-host=127.0.0.1
db-url-port=3306
db-url-database=keycloak_db
db-pool-max-size=10
db-pool-min-size=5

bootstrap-admin-username=admin
bootstrap-admin-password=admin

http-host=0.0.0.0
http-port=9080

# 有代理时用
# proxy-headers=xforwarded
http-enabled=true

#信任域名
#hostname=dev.keycloak.com
hostname=192.168.0.1
hostname-strict=false

# openssl方式生成的证书形式配置，分别公钥和私钥两文件
#https-certificate-file=/opt/keycloak/certs/tls.crt
#https-certificate-key-file=/opt/keycloak/certs/tls.key

# keytool方式生成的证书形式配置，一个文件
https-key-store-file=/opt/keycloak/certs/keycloak.jks
https-key-store-password=changeit
https-key-store-type=JKS

# https端口
https-port=9443

health-enabled=true
metrics-enabled=true


log=console
log-level=info

http-management-port=9090
http-management-health-enabled=true
```
## 配置服务

创建文件`/etc/systemd/system/keycloak.service`

```plain
[Unit]
Description=Keycloak Server
After=network.target

[Service]
Type=simple
User=root
Group=root
WorkingDirectory=/opt/keycloak
# 环境变量形式
# EnvironmentFile=/opt/keycloak/env/keycloak.env
ExecStart=/opt/keycloak/bin/kc.sh --config-file=/opt/keycloak/conf/keycloak.conf start --optimized
Restart=on-failure
LimitNOFILE=102642

[Install]
WantedBy=multi-user.target
```
## 启动服务

```plain
systemctl daemon-reload
systemctl enable keycloak
systemctl start keycloak
systemctl status keycloak
```
## 注意事项

### 开发模式

* 以`bin/kc.sh start-dev`形式启动

* HTTP 已启用

* 严格主机名解析已禁用

* 缓存设置为本地（未使用分布式缓存机制来实现高可用性）

* 主题缓存和模板缓存已禁用

### 生产模式

生产模式要求在启动时设置主机名并启用HTTPS/TLS连接。

* 以`bin/kc.sh start`形式启动

* 由于传输层安全性（HTTPS）至关重要，因此HTTP被禁用

* 设置主机名

* 需要 HTTPS/TLS 配置

**强制HTTP**

* 开启http：`http-enabled=true`

* 修改数据：`UPDATE REALM SET ssl_required='NONE' WHERE name='master';`

### 常见问题

#### SELinux 限制

报 status=203/EXEC，说明 SELinux 或隐藏字符 导致 systemd 无法识别脚本

>SELinux是什么
SELinux（Security-Enhanced Linux）是 Linux 内核的一个 强制访问控制（Mandatory Access Control, MAC）机制，作用是：

* 限制程序访问文件、端口、设备等资源的权限

* 即使你是 root 用户，也可能被 SELinux 阻止执行某些操作

* 用于防止被入侵后程序滥用系统资源

>解决方式
**临时方案**

```plain
# 将 kc.sh 设置为可执行脚本类型
chcon -t bin_t /opt/keycloak/bin/kc.sh
# 或将 SELinux 从 Enforcing（强制） 模式切换到 Permissive（宽容） 模式
setenforce 0
```
**永久解决**
* 创建自定义 SELinux policy，允许 systemd 执行 /opt/keycloak/bin/kc.sh

* 或者用 semanage fcontext 设置永久上下文：

```plain
# 安装 semanage
dnf install -y policycoreutils-python-utils
# 永久设置 /opt/keycloak/bin 下所有文件类型为 bin_t
semanage fcontext -a -t bin_t "/opt/keycloak/bin(/.*)?"
# 应用上下文修改
restorecon -R /opt/keycloak/bin
```
验证
```plain
ls -Z /opt/keycloak/bin
```
输出类似
```plain
-rwxr-xr-x. root root unconfined_u:object_r:bin_t:s0 kc.sh
```
* bin_t 类型表示 SELinux 允许 systemd 执行

* 重启 systemd 或机器后，这个上下文会保留
